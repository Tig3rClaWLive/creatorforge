CreatorForge Update – Admin, Creator-Profile und Download-Fix

Was geändert wurde:
- Creator-Verzeichnis stark ausgebaut:
  - Suche nach Creator
  - Avatar/Banner-Karten
  - Upload-Anzahl je Creator
  - Download-Anzahl je Creator
  - Spendenbutton sichtbar, wenn donation_url gesetzt ist
  - Social-Buttons nur für gespeicherte gültige Links
- Startseite zeigt Top-Creator jetzt nach Downloads/Uploads.
- Download-Detailseite zeigt beim Creator Upload-/Downloadzahlen und Bio.
- Upload speichert jetzt original_filename und mime_type.
- Download liefert richtige Dateinamen und Dateiendungen aus.
- Neue D1-Migration db/v5_file_metadata_creator_stats.sql hinzugefügt.

Wichtig: Diese ZIP enthält bewusst KEIN node_modules, KEIN .next und KEIN .git.

Einspielen:
1. Dateien aus der ZIP in deinen Projektordner kopieren und vorhandene Dateien ersetzen.
2. Im Projektordner ausführen:

   npm install
   npm run build

3. Falls original_filename und mime_type noch nicht in der Cloudflare-D1 sind:

   npx wrangler d1 execute creatorforge-db --remote --command "ALTER TABLE uploads ADD COLUMN original_filename TEXT;"
   npx wrangler d1 execute creatorforge-db --remote --command "ALTER TABLE uploads ADD COLUMN mime_type TEXT;"

   Wenn dort "duplicate column name" steht, ist das okay. Dann existiert die Spalte bereits.

4. Indexe optional anlegen:

   npx wrangler d1 execute creatorforge-db --remote --command "CREATE INDEX IF NOT EXISTS idx_uploads_user_status ON uploads(user_id, status);"
   npx wrangler d1 execute creatorforge-db --remote --command "CREATE INDEX IF NOT EXISTS idx_uploads_downloads ON uploads(downloads);"

5. Danach committen und pushen:

   git add .
   git commit -m "Expand creator profiles and fix download metadata"
   git push

Danach in Cloudflare Pages prüfen, ob der neue Deploy erfolgreich war.
